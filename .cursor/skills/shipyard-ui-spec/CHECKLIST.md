# Checklist（验收清单）

用于每次页面改造/新增页面时的最终检查，保证“好看 + 好用 + 一致”。

## 视觉一致性

- [ ] 页面使用 `NPageHeader`，标题层级清晰
- [ ] 同一页面只有 1 个 Primary CTA
- [ ] 间距遵循 4 的倍数（8/12/16/24/32）
- [ ] Card/Modal/Drawer 圆角统一（默认 8~10）
- [ ] 不在页面里硬编码随意颜色（颜色来自主题或语义化 type）
- [ ] **布局与间距用 UnoCSS**，避免散落 `style="..."` 或大量 `scoped CSS`

## 信息结构

- [ ] 重要信息可扫读（状态/环境/分支/耗时）
- [ ] 次要信息弱化（更小字号/更浅颜色）
- [ ] 超长文本有 ellipsis + tooltip（表格/列表）

## 状态处理（必须有）

- [ ] Loading：有 `NSpin` 或 Skeleton
- [ ] Empty：有 `NEmpty`（文案明确 + 可引导下一步）
- [ ] Error：有 toast + 可重试（如适用）
- [ ] 提交态：按钮 loading、禁用重复提交

## 交互与安全

- [ ] 危险操作二次确认（删除/解散/重置等）
- [ ] 表单校验可读（必填/格式/长度）
- [ ] 权限不足给出明确提示（403 文案）

## 可访问性（最小要求）

- [ ] 主要按钮可通过键盘聚焦
- [ ] 表单项有 label，错误信息可被感知

## UnoCSS（强制）

- [ ] 页面主要结构（布局/间距/排版）使用 UnoCSS utilities/shortcuts
- [ ] 如出现 `<style scoped>`，必须能说明其必要性（否则改回 UnoCSS）
